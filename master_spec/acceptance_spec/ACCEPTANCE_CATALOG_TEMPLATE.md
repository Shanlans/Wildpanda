# Acceptance Catalog Template

## Metadata
- Path:
- Owner:
- Last updated:
- Classification: `template-owned skeleton`

## Purpose
- Provide the reusable structure for repository acceptance catalogs.
- Keep category definitions reusable while requiring repository-instance profile entries.

## Template Rules
- The catalog structure is reusable.
- The listed profiles and their exact paths are repository-instance data.
- Each repository instance catalog should reference `ACCEPTANCE_PROFILE_TEMPLATE.md` as the profile-structure template source.

## Recommended Entry Fields
- profile id/name
- instance file path
- use case
- instance-specific notes
